CSS - Guide

O que é o CSS?

CSS é um acrônimo para Cascading Style Sheet. É um código para criar estilos no HTML. Básicamente o HTML é a estrutura e o CSS é a beleza, vem para transformar a estrutura em uma aplicação devidamente apresentável. Igual o HTML o CSS não é uma linguagem de programação.

Estilos de CSS:

O css pode ser adicionado no seu HTML de três formas, sendo a terceira forma a mais utilizada.

  1. CSS Inline - dentro do atributo style="color:red"
  2. CSS Interno - dentro do head criar a tag style
  3. Css Externo - criar o próprio arquivo CSS.

DevTools

Ferramenta quase obrigatória para que possamos estudar, compreender e utilizar o CSS no nosso dia a dia. Ele consegue nos mostrar exatamente todos os elementos de uma página junto com seus códigos HTML e CSS.

Cascata do CSS:

É importante saber que o estilo aplicado em sua página sempre será lido de cima para baixo e deve ser levado em consideração três fatores para a execução correta do estilo: Origem, especificidade e importância.

  • Origem - Inline > Interno > Externo
  • Especificidade - Id > Class > Elemento > Universal *
  • Important - Ao colocar !important depois do atributo, ele irá sobreescrever tudo.

Seletores do CSS:

Seletores servem para linkarmos o elemento HTML que voce deseja estilizar com o seu CSS do projeto.

  1. Direto em qualquer elemento tag HTML - h1{ }
  2. Através da Class - Múltiplo - .NomeDaClasse
  3. Através do ID - Único - #NomeDoID
  4. Efeitos globais - * { } - Tudo que está no arquivo
  5. Para alplicar mais de um seletor - .titulo, #paragrafo{}

Combinators

Através de combinadores podemos trabalhar para buscar e combinar seletores afim de aplicar uma estilização.

  • Descendent - busca um elemento dentro de outro.
    • Ao colocarmos por exemplo body article h2 {}, ele irá estilizar todos os h2 que estejam dentro de articles que esteja dentro do body.
  • Child - Identificado pelo sinal > entre dois elemtnos. Só seleciona o elemento que é filho direto do pai.
    • Ao colocarmos por exemplo body > ul > li , ele irá estilizar o primeiro UL que estiver após o body, e cada li seguinte. No caso acima, ele iria estilizar todos os UL que estejam dentro do body.
  • Sibling - Identificado pelo sinal de + entre dois seletores. Ele irá selecionar somente o elemento do lado direito que é irmão direto na hierarquia.
    • Ao colocarmos por exemplo button + button{} ele irá estilizar o botao que possua outro botao dentro da mesma hierarquia . ex: div>button*2
    • Ao colocarmos por exemplo h1 ~ p , ele irá selecionar todos os paragrafos que estejam na mesma hierarquia(abaixo da tag h1)

Pseudo-Classes

É um tipo de seletor que irá selecionar um elemento que estiver em um estado específico. Por exemplo: É o primeiro elemento dentro de uma caixa. O elemento que está com o ponteiro do mouse encima dele entre diversos outros.

  • first-child - Seleciona o primeiro filho de um elemento.
    • ao colocarmos por exemplo ul::first-child ele irá selecionar o primeiro item dentro da lista.
  • nth-of-type - voce escolhe qual elemento filho voce deseja selecionar.
    • por exemplo, ao colocarmos ul:nth-of-type(2) irá selecionar o segundo elemento dentro da ul.
  • Even/Odd (Pár e Ímpar) - seleciona itens pares ou itens ímpares
    • ao colocarmos por exemplo ul li:nth-child(odd) dentro de uma lista, ele irá estilizar apenas os itens em posições ímpares.
  • Hover and Focus - Personaliza ao usuário passar o cursor por cima do elemento e quando o elemento estiver focado (em caso de inputs text's).
  • Disabled e Required - Quando o imput for disabled por exemplo, voce consegue personalizar apenas quando estiver desativado. Da mesma forma se aplica para o required.
    • input:disabled {} ...

Box Model

O CSS trabalha com a idéia de caixas. Todas as caixas possuem seus limites (altura e largura). Com o tempo iremos perceber que praticamente tudo que encontramos nos layouts são construídos a base de caixas. Toda caixa possuirá seu posicionamento, tamanhos, espaçamentos, bordas, cores e aí por diante. Todos os elemtnos HTML são caixas.

  • Box-Sizing - Devemos ficar atentos ao tamanho de nossas caixas, que podem ser trabalhadas de duas formas:
    • content-box - é o que vem por padrão. Se voce estipular que o tamanho da caixa é de 50px, ele irá colocar o tamanho do conteúdo de 50px, e caso haja paddings, margins, etc, será somado a parte.
    • border-box - Diferente do citado acima, se definirmos o tamanho da caixa de 50px e essa caixa possua borda, ela automáticamente irá fazer o calculo para que a caixa como um todo seja de 50px.
  • Display-Block & Inline - Caixas podem se comportar com dois tipos de displays:
    • display block - ocupam toda a linha, colocando o próximo elemento abaixo, width e height são respeitados, padding margins e border irão funcionar normalmente.
    • display inline - elemento ao lado do outro, width e height não funcionam, somente valores horizontais de margin padding e border irão funcionar.
  • Margin - Externo - Espaço entre os elementos, geralmente definidos com três possíveis formas:
    • unidades exatas (px,em)
    • porcentagem (%)
    • auto - centraliza o objeto na horizontal
  • Padding - Interno - Preenchimento interno da caixa. Distancia entre a borda e o conteúdo.
    • unidades exatas (px,em)
    • porcentagem (%)
    • auto - centraliza o objeto na horizontal
  • Border-Outline

Posicionamento CSS

Quando falamos em posicionamento devemos pensar em como montar corretamente o layout de uma página na tela. Antigamente, esses layouts eram criados através de tabelas no html.(Tática não utilizada hoje em dia).

Nos dias de hoje, existem duas formas nas quais são construídos os layouts modernos.

  • FlexBox
  • CssGrid

Posicionamento no CSS talvez seja o tópico que mais possa gerar complicações no assunto, por isso é necessário atenção. Para controlarmos onde nosso elemento irá ficar, podemos usar de primeiro a propriedade Position.

Vale lembrar que por padrão, um elemento ficará embaixo do outro e ocupará toda sua linha até o final.

  • Static - fluxo padrão, um abaixo do outro.
  • Relative - ao aplicarmos, se torna possível aplicarmos cinco propriedades : right,bottom,left,top e z-index. Dessa forma podemos empurrar cada objeto setando seus pixels. Exemplo: position:relative top: 20px - irá afastar no top em 20 px.
  • Absolute - Também libera todos as cinco propriedades acima. Diferente do Relative, ele de certa forma faz o elemento ir para outra camada, jogando o elemento selecionado para um novo fluxo.
  • Fixed - Volta a mesma idéia de que ele sai do contexto, porém o elemento fica fixo na página.
  • Element Stacking :

Flexbox

Quando falamos em flexbox, falamos principalmente nos layouts das páginas e a forma com que ela é estruturada. O Normal flow é a maneira que os elementos block e inline ficam na página por seu comportamento padrão. Obviamente que o normalflow não deixa uma página harmônica e nem visivelmente apresentável e para isso que precisamos ir atrás de customizações.

Durante muito tempo foi utilizado o modelo em tabelas, nas quais através de uma tabela, cada campo recebe um determinado display, assim constituindo o layout do site. Este modelo NÃO é mais utilizado nos dias de hoje. Com a evolução foi passado a utilizar o modelo tableless, na qual passaram a implementar o float-left, etc, porém este modelo também não é utilizado nos dias de hoje.

Enfim, chegamos onde as páginas modernas são construídas no dia a dia, o FlexBox. O flexbox é uma caixa que se tornam flex, fazendo com que seus elementos dentro da caixa possam receber um melhor alinhamento, ordenação e tamanhos flexíveis.

Ao aplicarmos o dislpay:flex, automaticamente eles passam a ficar um ao lado do outro, se tornando mais fácil sua ordenação.

Terminologia:

  • Flex Container & Flex item - Para que o flex funcione, precisamos pegar um container maior para ser o flex container, e todos os itens dentro dele se tornarão o flex item. Este conceito de elementos dentro de outros elementos, é chamado de Nesting.
  • Axis - Eixo Principal x Cruzado - Ao trabalharmos com Flexbox precisamos sempre ficar atentos aos eixos, pois através do flex-direction podemos alterar o eixo, colocando da horizontal para a vertical por exemplo.
  • Flex Sizing - É o conceito do item ser flexível. Alterar a altuar e largura dos itens para preenchimentos dos espaços do flex container. Colocando Flex:1 por exemplo, todos os itens dentro do container passarão a ocupar a mesma porção dentro dele.

Propriedades do Flex-Container:

Ao colocarmos DisplayFlex dentro de um container, podemos alpica-las uma série de propriedades

  • Flex Direction: Com flexbox ou os itens ficarão na horizontal ou na vertical. Através do Flex Direction podemos alterar seu eixo, e podemos colocar os seguintes valores:
    • row - é o que vem no padrão.
    • row-reverse - Coloca de trás para frente 100% ao inverso.
    • column -
    • column reverse -
  • Flex-Wrap - O flexbox tem a capacidade de utilizar multilinhas, na qual cada linha será um novo eixo a ser criado. Por padrão, o flex vai fazer sempre o item se encaixar dentro do limite do seu flex. A não ser que coloquemos o Flex-Wrep. Caso ele tenha tamanho suficiente para ficará em uma só linha e caso contrário ele irá quebrando.
  • Flex-Flow - É um ShortHand do Flexbox, na qual colocamos o flex direction e o Wrap . Assim colocamos o FlexFlow direction + wrap. Ex: flex-flow: column wrap.
  • Justify-Content - Alinhamento e distribuição do container no eixo principal (horizontal). Para isso temos alguns valores a serem colocados
    • flex-start - padrão, um ao lado do outro.
    • flex-end - alinha ao final do container.
    • center - alinha ao centro do container
    • space-around - espaços por igual ao redor deles, inclusive entre os da ponta e o container.
    • space-between - espaço entre eles mas os da ponta ficam colados no container.
    • space-evenly - espaço por igual em cada um deles.
  • Align-Items - Alinhamento dos elementos no eixo cruzado (vertical). Para isso temos alguns valores a serem colocados
    • flex-start - deixa os elementos no inicio do eixo cruzado
    • flex-end - deixa os elementos no final do eixo cruzado.
    • center - deixa os elementos no centro do eixo cruzado.
  • gap - capacidade que temos em colocar espaço entre os elementos. Podendo ser colocados através de medidas fixas(pixels) ou flexiveis como porcentagens por exemplo. Exeplo: gap:4px, colocando 4px de distancia entre cada elemento.

Propriedades do Flex-Item

Assim como o flex-container tem suas propriedades para organizar todos os itens dentro dele, cada item também possui suas respectivas propriedades para que possamos estiliza-los.

  • Flexis-Basis - Serve para ajustar medidas dos elementos.
    • auto - vem por padrão, respeita o tamanho do conteúdo de cada elemento.
    • flex-basis: 25px - aplica 25 pixels para cada elemento. Também aceita %, etc.
  • Flex-Grow - Capacidade da expansão de um elemento para ocupar os espaços vazios sobrando de um flex-container. Através dele, colocamos como valor 1, 2, para que cada um vá pegando uma porção respectivamente.
  • Flex-shrink - Capacidade do encolhimento de um elemento.
  • Flex - Propriedade ShortHand igual temos no Flex-Flow, podendo em uma única linha aplicar todos os três comandos acima na ordem: Shrink->Grow->Basis.

Cores no CSS:

Como utilizamos o CSS para estilização de nossos projetos, então é através dele que conseguimos setar as cores da aplicação. Quando falamos de cores, falamos de color(textos), bg color (fundo de caixas), border-color para as bordas, entre outras.

Cores são muito mais importante do que pensamos, afinal, a harmonia visual de um site nos diz se iremos consumir o seu conteúdo ou não. Muitas das vezes isso acontece de forma irracional, sites estéticamente harmônicos atraem mais os usuários. Para mais informações sobre isso, estude mais sobre psicologia das cores e a UI.

No CSS3 existem quatro principais formas de definirmos valores das cores:

  1. Name - background-color: red
  2. RGB - background-color: rgb(122,0,140)
  3. HexaDecimal - background-color: #12C2F8
  4. Globais - inherit(herança), initial(cor inicial)

Agora que já sabemos como representar as cores precisamos também aprender a harmoniza-las. Você consegue se aprofundar no assunto estudando design/UI, mas segue abaixo algumas ferramentas e dicas de como criar uma paleta de cores harmônica.

  • Adobe Colors - Grande parte gratuita na qual nos ajuda a montar uma paleta eficiente e também a arrastar arquivos para extrair uma paleta baseado no arquivo, como uma logo do instagram por exemplo. Também pode nos fornecer paletas baseado em layouts mais utilizados na internet.
  • Paletton - Funciona de forma parecida com o adobe colors, porém nos permite uma pré visualização de um layout aleatório utilizando a paleta inserida nele.
  • Coolors - Divide sua tela em 05 partes e gera uma paleta aleatória, na qual ao apertar espaço ele vai mudando a paleta até chegar uma do seu gosto. Também é possível lockar alguma das cores, para as demais irem mudando e a que você gostou ficar travada.
  • ColorZilla Extensão do GoogleChrome que nos permite capturar a # de uma cor específica que você olhar em qualquer página na internet.

Bordas no CSS

Colocamos a propriedade Border com 03 informacoes distintas

  1. Primeiro colocamos os pixels ex: 5px
  2. Em seguida colocamos o tipo de borda ex: Solid, Dotted, Dashed,
  3. Por ultimo colcamos a cor da borda ex: blue
  4. Coloque tudo em uma linha: Border: 15px solid black
  5. border-radius: 5px - arredonda as bordas da borda.

Textos & Fontes

  • Color: blue - altera a cor do texto
  • font-size: O tamanho da fonte pode ser aplicado em medidas absolutas e medidas relativas. Exemplos:
    • Absolutas - cm, mm, in, px, pc
    • Relativas - em (relativa ao tamanho), ex (relativa a uma altura X), rem (relativo ao root), vw (relativo a viewport), %.
    • Por recomendação da W3C as mais apropriadas para uso são a PX e EM.
  • font-weight: Relacionada ao peso da vonte, podendo receber os seguintes valores:
    • Lighter
    • Bold
    • Bolder
    • Numérico - Ao invés das palavras acima, podemos colocar 100,200,300 por exemplo, para ir setando o peso da fonte.
  • font-family: Arial - Altera o tipo da fonte.
    • aqui voce pode colocar mais de uma separada por virgula, para caso nao haja a primeira. Ex: font-family: Arial, xxx, etc. Dessa forma caso no dispositivo ou navegador não ache a Arial ele irá aplicar a xxx, e caso não haja irá colocar a etc.
  • font-style- Altera o estilo da fonte, vindo como padrão o normal, podendo ser alterado para italic.
  • text-align: left, center, right, justify - alinhamento do texto.
  • text-align - Alinha o texto dentro do espaço que ele está.Podemos utilizar alguns valores:
    • center - centrilizado
    • right - alinhado a direita
    • justify - alinha o texto a esquerda e a direita.
  • text-indent - Em PX voce coloca a separacao do inicio do texto da borda.
  • text-decoration: none(sem decoracao), over-line(linha encima), line-trhough(linha cortada)
  • text-transform: uppercase(maiusculo), lowercase(minusculo),
  • letter-spacing: 10px - espacamento entre as letras.
  • word-spacing: 20px - distanciamento entre uma palavra e outra.
  • line-height: 10px - distanciamento entre uma linha e outra.
  • text shadow: 0px(dist direita) 2px(dist abaixo) 3px(fumaça) red(cor) - sombra no texto.

Fontes Externas:

Muitas das vezes iremos utilizar fontes customizáveis da internet na qual ela não estará instalada nos dispositos que irão acessar a aplicação. Para isso, igual nas imagens, podemos utilizar uma fonte hospedada em outro servidor, ou baixarmos e hospedar a fonte dentro do nosso próprio projeto.

Quando falamos de fontes externas, temos como referência o Google Fonts, que nos permite utilizar diversas fontes de seu servidor.

E para implementa-las na nossa aplicacão é muito fácil, podemos fazer isso via link(Html5) ou como @import(Css3). O LINK fica sempre dentro do HEAD, dentro do HTML5, e o IMPORT fica dentro do CSS no topo da página geralmente.

Já para implementarmos fontes baixadas, primeiramente precisamos ter essas fontes em nossa máquina, e para isso o site DaFont pode te ajudar a encontrar a fonte de seu agrado. Após o download, jogue a fonte para alguma pasta dentro do seu projeto. Em seguida, dentro do css use o código @font-face que ele já irá montar a estrutura para voce colocar onde af onte se localiza e setar o nome da fonte desejada.

Caso tenha ficado dúvidas, confira nesta aula gratuita do Curso em Vídeo.

Background

Através do Background podemos definir o fundo de alguma div, colocando duas propriedades mais utilizadas.

  • Background-color - colore a div em uma cor específica, mas também podemos criar degrades, por exemplo:
    • background-color: linear-gradient(to bottom), yellow, red; - Dessa forma, surge um degrade do amarelo para o vermelho de cima para baixo.
  • Background-image- Coloca uma imagem no fundo da div, na qual por padrão ele irá repetir a imagem para um lado e para o outro, de forma com que ele consiga ocupar 100% da ViewPort. Podemos alterar esse comportamento através da propriedade background-repeat.
    • background-repeat: no-repeat - Aplica apenas uma imagem sem repetir.
    • repeat X ou repeat Y - Repete apenas no eixo desejado.
  • Background-Position - Quando colocamos uma imagem maior do que o tamanho da div em que ela se encaixa, podemos posiciona-la da forma que aparecerá a parte desejada da imagem. Para isso usamos o bg position com as seguintes informacoes:
    • background-position: left top - centraliza na esquerda e para o topo, e assim podemos ir alterando da forma que quisermos para pegar a parte desejada.
  • background-size - Muitas das vezes iremos precisar reajustar o tamanho da imagem de fundo para que ela possa se adaptar ao certo no ponto desejado da sua aplicação. Como boa prática, é sempre indicado colocarmos uma cor de fundo no local onde irá receber a imagem de fundo, para que quando a aplicação estiver carregando a imagem,a tela não fique completamente em branco.
    • background-size: 1000px - voce seta em pixels o tamanho da imagem de fundo, porém não é muito indicado pois dependendo do tamanho do monitor do usuário, sua aplicação pode ficar quebrada.
    • background-size: 100% - adapta a imagem a 100% da VP, porém ela provavelmente ficará preta na parte inferior.
    • background-size: contain - faz com que a imagem seja 100% exibida independente do tamanho da tela.
    • background-size: cover - cobre a tela inteira mesmo que para isso tenha que cobrir levemente a imagem.
  • background-attachment - Aqui podemos criar vínculos no CSS. Podemos utilizar basciamente duas propriedades:
    • background-attachment: scroll - é o padrão, rolou o conteúdo o fundo rola junto.
    • background-attachment: fixed - Ele pausa a imagem de fundo e apenas o conteúdo rola, muito usado para o efeito parralax nos sites.
  • ShortHand - E por fim, podemos resumir todos os itens acima em apenas uma linha, seguindo a ordem:
    • background: color > image > position > repeat > size > attachment

MediaQueries & Responsividade

Nos dias atuais conseguimos consumir conteúdo na internet em diversos dispositivos com diversos tamanhos de tela diferentes, como por exemplo abrir um site em um Macbook, Ipad, Iphone.Para que o site fique harmônico em diversos tipos de tamanhos, precisamos usar a mediaquerie para que ele fique responsivo e adaptativo. Também pode ser utilizada para mais coisas como por exemplo para a impressão de uma matéria em uma página. Através da mediaqueries podemos excluir os menus e demais coisas do site que não são desejáveis na impressão.

Ao criarmos o LINK do CSS dentro do Head do HTML, podemosc colocar o atributo media="".

  • media="all" - Css responsável para estilização geral da aplicação.
  • media="screen" - CSS responsável pela estilização da tela.
  • media="print"- CSS responsável pela estilização da impressão.
