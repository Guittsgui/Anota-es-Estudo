HTML5 - Guide

O que é HTML?

Primeiramente devemos saber que HTML não é uma linguagem de programação e sim uma linguagem de marcação de texto. Basicamente é uma forma de escrita através de elementos, na qual é interpretada pelo seu navegador. Essas marcações são feitas através de TAGS. Cada tag possui sua função, suas instruções e algumas possuem seus respectivos atributos. Atributos servem como informacoes extras para colocarmos no elemento ou configuracões necessárias para que o elemento funcione corretamente como links ou imagens por exemplo.

Anatomia do Documento

Para escrevermos um documento em HTML é necessário saber que ele precisa ser feito dentro de uma estrutura HTML. Se voce utiliza o VisualStudio como editor de código, basta criar um arquivo qualquer com a extensão .html, e dentro dele digitar ! e dar enter. Dessa forma a estrutura será montada automátiacmente.

Head e Body

Ao construir sua primeira aplicação HTML, voce deve perceber que a estrutura é dividida entre head e body.

  • HEAD - Tudo que for escrito dentro do head não será exibido no navegador. Ali ficam as configurações de linguagem do documento, títulos, links para arquivos externos, mecanismos de busca entre os navegadores, dentre diversas outras funções.
    • Meta - Tag especial que dfine metadados.
      • charset=UTF8 - define padrão de caracteres especiais.
      • name/content - define as partes visíveis do site e sua adaptação em outros devices menores.
    • Favicon - Pequenos ícones ao lado do título na aba do navegador.
    • MetaSEO: optimização da página para motores de buscas.
      • author/content - define o autor e sua propriedade.
      • description/content - Texto ao aparecer em sua pesquisa no google.
      • robots/content - Especifica o que voce quer que o robo do google faça.
  • BODY - Tudo que for escrito dentro do body é o que vai ser visto dentro do seu projeto. Ali começa a construção de sua aplicação em si.

Atributos Glogais

Atributos globais são atributos que podem ser aplicados em qualquer elemento, porém nao necessáriamente cada atributo global fará sentido em todas as tags existentes.

  • Class/ID - aplica uma classe a um elemento.
  • Contenteditable - permite editar o conteúdo na página.
  • data-qualquerNome - muito utilizado para lógicas no JS.
  • hidden - esconde uma tag.
  • style - forma de colocar CSS Inline.
  • tabindex - altera o foco ao apertar tab em cada item da página.
  • title - define um título ao deixar o mouse parado.

Tags Semanticas no HTML

O HTML5 é baseado em tags semanticas. Tags que atribuam significado para o conteúdo e ao navegador . Essas são de extrema importância para a construção de aplicações modernas e otimizadas.

Aqui podemos ver tags para estruturacão do site:

  • Header - tudo que fica no cabeçalho do site
  • main - conteúdo principal do site
  • footer - conteúdo do rodapé do site
  • nav - conteúdo dentro de menus do site
  • section - conteúdo de alguma sessão do site
  • article - conteúdo dentro de um artigo no site.
  • aside - conteúdo extra em algum site.

Links e Âncoras

Através dessa tag A é que podemos navegar entre páginas e informações. Sempre que utilizamos, é obrigatório utilizar o atributo HREF, pois ele que irá nos dizer para onde esse link irá nos levar, podendo ser uma outra página, ou até mesmo para um determinado local dentro da mesma página.

Para que a navegação ocorra sem erros, é necessário saber com exatidão a URL da página desejada (em caso de páginas externas), ou o caminho entre os diretórios do seu projeto.

  • target="_blank" - Faz com que você não saia da sua página e sim abre outra aba no navegador. (Muito utilizado para quando saimos da nossa aplicação).
  • title="xxxxxx" - ao passar o mouse por cima do link, aparece o título desejado.
  • Podemos marcar um determinado local do site com id ou classe, e através do link jogar para lá.
    • id="aqui" - aheref="#aqui" - será rolado para o ponto marcado.

Também é possível fazer downloads via links, para isso colocamos o caminho do arquivo dentro do href e completamos com alguns atributos:

  • download="nomedodownload.extensao"
  • type="application/pdf" onde cada tipo de arquivo voce pode conferir Clicando Aqui.

Imagens

Provavelmente todas as aplicações da WEB que iremos desenvolver serão usadas imagens. Precisamos saber inicialmente sobre direitos de imagens. Precisamos tomar cuidado com os direitos autorais. Você não pode utilziar qualquer imagem encontrada na internet para seus projetos profissioanais. Para isso, o prórpio google possui um filtro sobre os direitos de suas imagens. Existem alguns sites próprios para imagens com os direitos autorais autorizados como o Unslash, Pexels,entre diversos outros.

Quando trabalhamos com imagens em sites, é fundamental entendermos sobre o peso das imagens e suas respectivas dimensões, afinal, isso irá impactar diretamente na velocidade em qual sua aplicação será processada. Muitas das vezes precisaremos alterar seu formato, suas dimensões e para isso podemos contar com a ajuda de um software gratuito e de fácil manuseio chamado Gimp.

Aprofundando no assunto, precisamos conhecer os principais formatos de imagens utilizados na WEB, que são inúmeros, porém podemos dar foco nos dois mais utilizados que são o JPEG e o PNG.

  • JPEG - Imagem mais compactada, fazendo com que ocupe menos espaço.
  • PNG - Permite transparência.

Ao trabalharmos com imagens, também precisamos ficar atentos ao seu tamanho, que obviamente quanto maior for, mais lento ficará o seu site. Não existe uma regra definida para isso, mas vale destacar que uma imagem, 1.500/1200 de largura já é o ideal/máximo praticamente para se trabalhar em um site. Vale ressaltar também, que a resolução fica tranquila entre 70/50.

Finalmente saindo da teoria e indo mais para a parte de código, veremos algumas tags importantes ao trabalharmos com imagens, mas antes disso, saiba que podemos adicionar imagens dentro do nosso próprio projeto, que irá subir junto com o servidor, ou então podemos utilizar urls de imagens hospedadas em demais servidores, tornando a nossa aplicação mais leve, porém caso o servidor onde a imagem caia, nosso site irá ficar sem a imagem também.

  • img- tag para adicionar a imagem na qual utilizamos dois atributos no mínimo:
    • src - onde voce coloca o diretório/link da imagem
    • alt - caso sua imagem não carregue, mostrará uma alternativa.
  • picture - Colocamos para abraçar toda a imagem, para que possamos setar o seu tamanho máximo e mínimo.
  • figure - figcapture - jeito semantico de colocar texto visível a imagem.
  • alt - coloca uma alternativa caso a imagem nao carregue
  • title - informa um texto ao voce deixar o cursor parado encima da imagem.

Imagens Dinâmicas

Para criarmos imagens dinâmicas via HTML, precisamos utilizar a tag Picture vista acima, pois através dela, poderemos colocar vários sources. Através desse source, setamos o min/max width de forma que assim possamos por exemplo: Abra a img X, SE a tela máxima for menor que 800px, abra a imagem Y e se a tela máxima for menor que 400px, abra a imagem Y. Lembre-se que para funcionar deve seguir uma ordem do maior para o menor ou vice-versa, mas sempre em ordem.

Caso tenha ficado confuso, basta reassistir a esta aula grátis do Curso em Vídeo clicando aqui.

Videos

Quando trabalhamos com vídeos, podemos utilizar quatro formatos: MP4 , M4V , WEBM , OGV. Os vídeos podem estar hospedados dentro do projeto do seu site (que o faz ficar mais lento), ou em algum repositório externo como o Youtube ou Vimeo por exemplo, tornando seu site mais rápido, porém quando esses repositórios cairem, seu site ficará sem vídeo.

  • video - tag para embrulhar todo o player.
  • source - tag para informar o local do vídeo.
  • type - para informar o tipo de vídeo adicionado.
  • autoplay - já inicia tocando o vídeo
  • preload - auto(já baixa o vídeo) - metadata(o básico).
  • loop - repete o vídeo.
  • poster - coloca a imagem de fundo antes do vídeo começar

Dica: Utilize o Software HandBrake para converter o formato dos seus vídeos.

Video Iframes

Utilizamos a tag iframe quando desejamos trazer um vídeo hospedado em outra plataforma, como o youtube ou o vimeo por exemplo.

Iframes não servem apenas para vídeos. Iframes são janelas que abrem dentro da sua página, fazendo com que voce possa abrir um site ali por exemplo. Para maiores informações consulte a esta aula gratuita do Curso em Vídeo.

Audios

Quando trabalhamos com audio, podemos utilizar três formatos: mp3, wav ou OGG. Em geral, as tags são basicamente as mesmas que existem nas tag video citadas acima.

  • audio - tag para embrulhar todo o player.
  • source - tag para informar o local do audio.
  • type - para informar o tipo de audio adicionado.

Forms

Parte de extrema importância dentro do HTML, pois sempre que formos construir aplicações na qual o usuário interaja com a mesma, usaremos formulários.

  • Form - principal tag para abrirmos e colocarmos tudo que ficará dentro do formulário.

    • Dentro delas, colocamos determinados atributos:

    • id - identificação do mesmo.
    • action - para onde ele será submetido.
    • method - (GET/POST).
  • Label - Serve para identificar e associar uma ou mais tags de entrada, acessibilidade. Ao clicar no label, voce automaticamente envia o cursor para o input. O for do label precisa estar linkado com o id do input para que funcione
  • Button - Representa um botão. É usado primariamente para enviarmos formulários. Ele é estilizado por padrão por cada navegador. Possui sempre alguns atributos padrões:
    • type - submit/reset/button
    • autofocus, disabled
    • name, value
    • form
  • Datalist -

Tags de Entrada de Dados

  • Input name id- Tag mais utilizada ao falarmos de forms, sempre vem acompanhada por diversos atributos citados abaixo.
    • type - define o tipo de texto do input.
    • autocomplete - busca no navegador algum tipo de dado usado frequentemente de acordo com o tipo do input.
    • autofocus - quando a página for carregada, o cursor ficará piscando no ponto desejado.
    • disabled - deixa desbalitado o campo selecionado.
    • form - Caso o imput esteja fora do form, ele linca o campo a algum formulário
    • required - não permite que o formulário seja enviado sem o campo estar preenchido (validação apenas do front.
    • placeholder - deixa uma mensagem pré digitada dentro do input do tipo: "Digite seu Email

Types of Inputs

  • Password - Funciona igual um text, porém com o texto escrito em ****. O segredo é que através dos atributos conseguimos controlar.
    • min/maxlength="" - define o tamanho da senha minima e maxima.
  • Email - Espera e valida se o usuário digitou um email
  • URL - Epsera e valida se o usuário digitou uma URL.
  • File - Escolhe um ou mais arquivos para enviar no formulário. Para funcionar o método do formulário precisa ser POST e também um enctype do tipo multipartform-data.
    • accept - Seta os arquivos que serão aceitos como por ex .doc , .mp3, .jpeg ou image/* que aceitaria todos os tipos de imagens..
    • multiple - permite enviar mais de um arquivo.
  • Color - Permite que voce selecione uma cor e no value será enviada uma cor RGB.
  • CheckBox - Caixas que podemos marcar e assim selecionando um valor. Caso nao selecione nenhuma, nenhum valor será enviado. Em caso de marcado, caso possua um value ok, caso contrário o backend irá receber como On.
  • Radio - Diferente do checkbox, ele é projetado para selecionar uma única escolha dentre um grupo de opções.
  • Hidden - Campo invisível ao usuário, leitores de tela não irão perceber esse campo. Utilizado caso queira por exemplo enviar por debaixo dos panos o horário que o formulário foi enviado.
  • TextArea - Serve para quando queremos colocar textos com mai sde uma linha.
  • Select/Option - Serve para nos informar um menu de opções para o usuário selecionar (Options).
    • Dentro da tag select, colocamos as options juntamente com seu atributo value que é o valor que irá para o backend.
  • Search - Campo de busca. Funciona igual o text, mas dependendo do seu navegador ele pode se comportar diferente.
  • Number - Funciona igual o text, porém só é aceito caracteres do tipo numbers.
  • Range - Nos dá um controle para selecionar um valor numérico através de um Slider.
  • Date - Campo que permite selecionar uma data (yyyy-mm-dd).
  • datetime-local - Além de dia mes e ano também trás o horário.
  • month - seleciona apenas o mes
  • week - seleciona qual a semana do ano
  • time - seleciona um horário.

Sempre lembre que formulários gigantes desanimam o usuário. Antes de começar a codar, pense cuidadosamente na UX do projeto.
