JavaScript Básico

Aqui damos início a nossa primeira linguagem de programação de fato. Através do Javascript iremos começar a dar funcionalidades e ações para as nossas aplicações. Através dele nossas páginas deixam de ser estáticas.

Em sua história, o javascript surgiu com o intuito de rodarmos programação no navegador mas hoje em dia usamos o JS tanto no Front-End (ReactJS) quanto no backend (NodeJS) quanto no mobile (ReacNative) quanto em desktops (ElectronJS). Ou seja, usamos JS para tudo. Mas para darmos passos grandes e entrarmos em frameworks, precisamos ter consolidado bastante os seus fundamentos da linguagem. De nada adianta aprender o framework sem antes saber dominar a linguagem em si.

Sintaxe

É a forma na qual escrevemos texto de uma maneira correta. Toda linguagem possui sua sintaxe. Boa parte dos erros dos iniciantes são causados pela sintaxe. Um simples ponto dentro do universo JS fora do seu devido lugar, pode fazer com que a aplicação inteira pare de funcionar.

Maneiras de executar o JavaScript

Existem basicamente três formas que veremos aqui de executar o JS.

  • Navegador - Podemos executar dentro do próprio browser apertando a tecla F12. Assim abriremos o DevTools onde encontraremos o Console, onde poderemos rodar nossos códigos.
  • CodePenIO - Site externo que nos proporciona um ambiente HTML CSS E JS. Ótimo para estudos de forma clara e limpa.
  • VSCode - Podemos abrir uma tag script dentro de qualquer arquivo HTML. E entao basta abrirmos ele no navegador e ver ele rodando.
    • Como boas práticas, geralmente é indicado criar um arquivo SCRIPT externo, e em seguida linkamos da mesma forma com qual fazemos com o CSS, utilizando a tag script. Vale ressaltar que os arquivos no editor são lidos de cima para baixo. Dito isso, geralmente ou ele é colocado dentro do Head, ou no final do Body. Dentro do Heads seria a melhor opção, porém se o seu código JS for muito grande, ele só irá carregar a página depois que todo o JS estiver carregado, não trazendo uma boa experiência ao usuário. Já no final do body, passa para o usuário a impressão de sua aplicação ser mais rápida.

Comentários

Para realizarmos comentários em um ambiente JS temos duas formas possíveis:

  • // - Aqui é utilizado quando queremos comentar apenas uma linha.
  • /* - Aqui abrimos a "tag" , e tudo ficará comentado até o final ou até seu fechamento */.

Variáveis & Escopo

São nomes simbólicos para receberem valores. Valores estes vistos acima como tipos de dados. Existem tres palavras reservadas para criarmos variáveis. Antes de falarmos da diferença entre as três, vale a pena entendermos o que é Escopo. Escopo determina a visibilidade de alguma variável no JS. Nem sempre as variáveis estão visíveis em toda a estrutura do JS. (Hoisting do inglês elevação).

  • Var - Var é global e local, pode funcionar fora de um escopo de código. O JS por debaixo dos panos puxa a variavel de qualquer outro escopo e joga ela lá para cima, fazendo sua declaração.
  • Let - Diferente do var o let é local, só funciona dentro do escopo na qual foi criada. Aqui não existe a elevação (Hoisting). Caso dentro de um escopo criemos uma variável com o mesmo nome de uma já existente no escopo global, ela será tratada como variáveis diferentes. Agora, caso usemos dentro de um escopo, uma variável global, ela será atribuida normalmente.
  • Const - Também é uma variável local, assim como o let, só funciona dentro do escopo que existe. Aqui é uma variavel imutável.

Concatenando e Interpolando

São táticas utilizadas para misturar frases com variáveis. Por exemplo: voce tem a variavel nome e idade. E em algum momento quer digiar : Olá, me chamo nome e tenho idade anos. Para isso usamos as duas táticas:

  • Concatenando : console.log("Olá, me chamo" + nome + "e tenho " + idade + "anos").
  • Interpolando: console.log(`Olá me chamo ${nome} e tenho ${idade} anos `).

Funções

Funções são trechos de código que precisam ser declaradas, e após isso, sempre que forem chamadas elas irão executar o seu respectivo trecho de código.

Function:

  • declarar - function minhaFuncao(){}
  • chamar - minhaFuncao()

FunctionExpression or Function Anonymous

  • declarar - const sum = function(){}
  • executar - sum()

Arrow Function:

  • const sum = () => {}
  • sum()

CallBack Function - Significa chamar de volta. É basicamente uma função que é passada por parâmetro para outra função.

Function Constructor.

xx

Parametros: Dentro desses parenteses nós conseguimos enviar e receber informações (variáveis & dados) para a função. Quando os dados são enviados na chamada são chamados de argumentos, e quando são recebidos na função são chamados de parâmetros.

Retorno de Valores: Funções podem ou não retornarem algo. Quando quisermos que retorne, utilizamos a palavra chave return.

Function Hoisting: Ao declararmos uma função de forma tradicional, ela pode estar em qualquer lugar do escopo global, que irá funcionar. Ao declararmos uma função anônima, ela nao irá sofrer o hoisting, logo, precisa ser declarada antes de ser chamada.

Manipulando Dados

  • (String(number)) - transforma um número em uma string.
  • (Number(string)) - transforma uma string numérica em um número.
  • string.length - calcula quantos caracteres existem em uma string.
  • number.toFixed(2) - um número quebrado passa a ter apenas duas casas decimais.
  • number.replace(".",",") - um número quebrado passa a ter vírgula no lugar do ponto.
  • string.toUpperCase() - põe toda uma string em letras maiúsculo.
  • string.toLowerCase() - põe toda uma string em letras minúsculas
  • let myArray = string.split(" ") - Cria um array, separando uma string por espaço ou por qualquer outro caracter desejado.
  • let fraseNova = myArray.join(" ") - Junta todo o array em uma só frase, colocando um espaço ou qualquer caracter desejado em uma nova string.
  • string.includes("vodka") - boolean que retorna true or false caso dentro da sua string contenha a palavra desejada. PS: CaseSensitive.
  • Array.from(string) - pega uma string e cada letra dela vira um elemento do array.

POO em JS

Aqui entraremos mais na parte de código da POO no universo JavaScript. Você pode ir para nossa aba de POO ou de PFunc para ver mais sob suas teorias. No JS podemos utilizar a POO através de classes, ou então de funções construtoras. Abaixo iremos abordar mais sob elas e qual usar em cada situação específica.

Classes - POO

Classes funcionam como uma espécie de Templates, ou seja, como o nosso objeto tem que funcionar. Nada nas classes de fato funcionam, ela irá passar a funcionar assim que um objeto for criado/instânciado. Todas as classes possuem básicamente duas coisas: Atributos e Métodos(). Para criarmos uma classe é fácil, basta colocar:

  • class Person{}
    • Por padrão colocamos a primeira letra maiúscula.

Logo em seguida é necessário criar uma função construtora. É uma função que será executada sempre que instanciarmos um objeto de uma classe, nesse caso, pessoa. Para isso, recebemos as propriedades por padrão, e dentro dela colocamos this.name = name;. O this, é um ponteiro, que significa "isto". No JS ele aponta para o próprio objeto que estamos criando. Se criarmos o João e a Maria, cada this apontará para seu respectivo objeto. Se quisermos colocar algum atributo já setado por padrão dentro da classe e fora do construtor(não vai receber externo), basta acima da função construtora colocar por exemplo: raça = 'humana', sem precisar declarar a variável ou algo do tipo.

  • constructor(name, age) {this.name = name ; this.age = age ;}

Após criada nossa Classe, para começarmos a utiliza-la precisamos realizar a Instância do objeto.

  • let pessoa = new Pessoa("Joao",30)
    • Nessa linha estamos instânciando um objeto pessoa através da classe Pessoa.

Além dos atributos, cada classe também pode ter os seus métodos(ações). Por exemplo, dentro da classe pessoa podemos criar a função "sayMyName". Para isso embaixo do construtor colocamos:

  • sayMyName(){console.log(`Olá meu nome é ${this.name} !`)}

Getter and Setters

Getter and Setters - É um conceito muito poular em POO, na qual o GET nos permite pegar o valor de um atributo e o SET nos permite SETAR o valor de algum atributo. Para isso criamos duas "funções" para isso.

Getter

  • get name(){return this.name}
    • caso tenha sido passado por parâmetro no construtor
  • get age(){return _age}
    • caso tenha sido setado na mão dentro da classe acima do construtor.

Setter

  • set name(name){this.name = name}
    • Pega por parâmetro o nome informado e altera.

Herança

Uma classe pode extender outra classe, ou seja, podemos "pegar" os atributos e métodos de uma classe. Por exemplo. Temos a classe pessoa com nome, idade, etc. Quando criamos uma classe Estudante por exemplo, ela pode extender a classe pessoa, para tambáem ter nome, idade, e também acrescentar os atributos apenas do estudante. Dessa forma evita códigos repetidos.

Para acontecer, primeiro colocamos a palavra extends. ex:

  • class Student extends Person{}

E a classe Student funciona como uma classe normal. Lá voce irá colocar seus atributos e métodos. Vale lembrar que para puxar todos os "dados" da classe pai, é necessário colocar a palavra super(name,age), dentro do construtor, para que ela possa executar o construtor do pai dentro dela.

Factory

Aqui é apenas um conceito que não está internamente no código em si, e sim na teoria. Ele é um conceito que significa fábrica. Em classes, o conceito de Factory, é fazer uma função construtora.

  • funciton createPerson(name){let p = new Person(name); return p;}

Através dessa função, logo em seguida podemos criar uma variavel e já atribuir sendo criada, por exemplo:

  • const p1 = createPerson("joão")

Dessa forma criamos uma Factory para a classe Pessoa. Através da função, ela fica mais organizado na parte em que formos instânciar em si. Digamos que é uma boa prática para um código mais "clean".
