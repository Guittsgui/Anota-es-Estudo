Node-JS - Basic

Para que serve ?

No início o javascript era utilizado apenas para rodar em navegadores. Com o passar do tempo o seu poder foi ficando cada vez mais forte devido a diversas tecnologias que foram surgindo em seu universo. Uma dessas tecnologias é o NodeJS. Ele nos permite desenvolver back-end, front-end, micro-services, API'S, tudo isso com o Javascript.

E quais as vantagens de utilizar NODEJS ? Uma das suas maiores vantagens é sua alta velocidade e escalabilidade. Outra grande vantagem que ele utiliza o JS, fazendo com que inciantes não precisem aprender outra linguagem de programação para desenvovler o back-end, e sim podem utilizar o JS, linguagem já utilizada no Front.

Mas, afinal, o que é o NODEJS? É impossível rodar JavaScript sem que tenhamos um ambiente próprio para isso. Por isso surge o Node. Node é um JS Runtime Enviroment, ou seja, é um ambiente que executa o JS em um ambiente completamente exclusivo a ele. Vale ressaltar que o NodeJS não é um framework como o React,Vue,etc.

E como ele funciona? O node funciona com base no Event Loop, ou seja, como o node é single threaded ele não bloqueia nada, trabalha de forma assíncrona, fazendo com que o Node trabalhe de maneira muito mais rápida que o Apache por exemplo.

Depois de já sabermos tudo sobre o que é, agora chegou a hora de instalar, para isso, acesse o seu site NodeJS e realize o download.

Variáveis Globais

Módulos

Node JS é baseado em módulos. Através da função required podemos utilizar diversos módulos dentro do nosso arquivo JS.

NPM - Node Package Manager

É um gerenciador de pacotes do node. Aqui iremos falar muito de dependências, pacotes, módulos etc. O primeiro comando que precisamos saber para começar a utilizar o NPM é como criar um novo pacote :

  • npm init || npm init -y (sim p/ tudo)

Utilizando este comando teremos acabado de criar nosso package.json. Além de criarmos, também podemos instalar módulos de terceiros, para isso utilizamos o comando:

  • npm i nomeDaDependência || npm i nomeDaDependência -D (dependencia de desenvolvimento)

Após instalarmos uma dependência de terceiros perceberemos que irá surgir o node_modules e o package-lock.json em nosso projeto. O node_modules, é onde fica todas as dependencias das dependencias que estalamos. Ao instalar uma dependência X, ela irá depender de uma Y, que poderá depender de uma Z. Tudo isso ficará localizado no nodule_modules.

Já o package-lock.json é um arquivo que basicamente nós não mexemos em nada, ele serve para mapear todas as dependências que temos em nosso projeto. Ele faz o projeto ficar mais rápido com esse mapeamento.

Também é possível instalar dependencias de forma global, ou seja, não estará instalado apenas em nosso projeto, e sim na máquina em geral, para todos os futuros projetos.

  • npm i nomeDaDependência -g
  • npm unistall nomeDaDependência -g

Outra coisa que faremos constântemente é atualizar versões de projetos. Para isso utilizaremos alguns comandos:

  • npm outdated - mostra todos os pacotes, suas versões mais desejadas etc.
  • npm upgrade - atualiza todos os pacotes para as versões desejadas.
  • npm i nomeDaDependência@latest - atualiza o pacote específico para ultima versão

E para finalizar, também será de extrema importância aprender a excluir dependências

npm unistall nomeDaDependência

NodeJS Events

O módulo de eventos já faz parte do core do NODEJS, e podem ser muito úteis como por exemplo: Alguém se inscreve em seu canal do Youtube, isso pode disparar um evento. (Exemplo ilusório). para isso, utilizamos o require(events), para que possamos começar a utiliza-los.

  • Event Emitter - É uma função que pegamos para a partir dela passar a emitir eventos.
    • const ev = new eventEmitter()
  • ev.emit('SaySomething')
  • ev.on('SaySomething')
