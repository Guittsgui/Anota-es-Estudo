Git & Github

O que é Git ?

Git é um sistema que nos possibilita realizar o controle de versão de nosso projeto. A cada modificação que ocorra em seu software, o git irá detectar isso, e assim podemos ir "salvando" diversas versões do nosso código. O git nos permite também retornar a versões antigas de nosso código. É como se fosse uma história, uma timeline. Outra vantagem, é que podemos criar novas branches, que seriam novas timlines.

Agora que já sabemos o que é, voce pode clicar aqui para baixar em sua máquina. E antes de começarmos lembre-se sempre do comando git help em seu terminal, lá será possível ver diversos comandos.

Comandos Iniciais no GIT

  • git initi - Inicia um repositório git dentro do seu novo projeto.
  • git status ::: monitora os arquivos que ainda nao estão mapeados no git..
  • git add . ::: adiciona todos os arquivos a zona de monitoramento.
  • git commit -m "xx" ::: realiza o commit com uma mensagem.
  • git commit -am "Juntou o ADD e o COMMIT no mesmo comando"
  • git log ::: mostra a lista dos commits realizados no projeto
  • git diff ::: mostra as alterações que ocorreram no projeto
  • git diff staged ::: mostra as alterações que estão no staged.

Desfazendo Mudanças

  • git restore arquivox - Restaura um arquivo modificado.(Antes de ir pro GIT ADD).
  • git restore --staged arquivox - Retira um arquivo do staged (.ADD)
    • Pode substituir arquivo específicio (arquivox) por . caso queira desfazer de todos.
  • git commit --amend - permite realizar alterações no último commit.

Trabalhando com Branchs

  • git branch ::: mostra a lista de branchs e a atual.
  • git reset xxxx idcommit ::: volta para uma determinada branch, podendo ter três formas diferentes:
    • -soft volta para o commit desejado, com as modificações presentes e não commitadas. Vai voltar para antes do commit.
    • -mixed basicamente faz a mesma coisa porém não estará preparado para commitar. A diferença é que as modificações devem ser add denovo.
    • -hard volta de forma definitiva para o commit desejado. Tudo feito na frente será perdido de vez.
  • git branch nomeNovoBranch ::: cria uma nova branch com o nome desejado.
  • git checkout nomeDaBranch ::: sai da branch atual e vai para a branch desejada.
  • git push -u origin main - envia para o github.

GitHub

Como vimos, o GIT é um gerenciador do histórico do seu projeto em sua máquina local. Quando pensamos em tirar nossos projetos da máquina local e enviar para uma plataforma remota, o GITHUB (e várias outras) entram para solucionar este problema. Utilizamos o GITHUB para ser nosso repositório remoto. Em resumo, podemos utilizar o git sem o GITHUB, mas não podemos utilizar o GITHUB sem o git.

Agora que já sabe mais ou menos o que é, para começar a utilizar você precisa criar uma conta no GitHub.

Como primeiro passo, precisamos aprender a criar repositórios dentro do gihub, para futuramente, conectarmos nosso projeto local a esse repositório. Para criar um repositório, basta logar no github, e no canto superior direito clicar em + -> New Repository.

Agora que já sabemos GIT e já possuímos nosso repositório remoto criado, podemos conecta-los e de fato passar a usar todo o poder do GIT + GITHUB.

Chave SSH - Autorizando Máquina

Para que possamos conectar nossa máquina com o github, precisamos realizar uma espécie de autenticação da máquina desejada com os servidores do GitHub. Para cada máquina que quisermos conectar, será necessário fazer uma autorização. Para realizar isso, clicamos na parte superior direita -> Settings -> SSH and GPG keys -> New SSH Keys. Chegando nesse ponto, abrimos nosso terminal e digitamos o seguinte código:

  • ssh-keygen -t rsa -b 4096 -C "emailDoGithub@gmail.com"

Em seguida damos ENTER, ENTER. Agora a chave SSH já estará criada. Geralmente ela será colocada dentro da pasta usuário. Basta irmos lá, pegarmos a PUBLIC, e arrastar para uma aba vazia dentro do próprio editor de código, nos possibilitando visualizar a chave escrita(Será vários e vários números e letras encriptografadas). Agora voltamos para a página do GITHUB, colocamos o título (Forma de saber qual máquina que é EX: Macbook Principal), e embaixo colamos essa chave. Pronto, seu dispositivo estará autorizado a enviar/receber do GitHub.

Caso queira ver o passo a passo de forma simplificada, basta acessar esta parte da Documentação do GitHub.

Linkando GIT com GITHUB

Agora que nossa máquina já está autorizada, podemos começar a usar a mesma. Para isso criamos um repositório geralmente em branco (Sem Readme,GitIgnore). Logo em seguida ele nos dará o passo a passo "Or push an existing repository from the command line". Então basta irmos na pasta principal do nosso projeto que queremos enviar para nuvem (tenha ele sido criado agora ou sendo antigo), e digitarmos os comandos na página do git.

Modificando Local e Enviando p/ Remoto

Trabalhando com git & github, o que mais iremos fazer é enviar arquivos do local para o remoto. Para isso basta seguir o passo a passo antigo normal já estudado em GIT. Git status -> Git ADD . -> Git Commit -m "". Após já estar devidamente commitado em nosso repositório local ele estara pronto para ser enviado para o nosso repositório remoto. Para isso basta executar o comando:

  • git push origin main, ou então apenas git push
