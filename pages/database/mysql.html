MYSQL - Banco Relacional

O que é um banco de dados?

Banco de dados é um repositório sistêmico de informações, ou seja, uma forma organizada de armazenar diversos tipos de informações. Bancos de dados podem ser relacionais ou não relacionais. Neste caso, estudaremos os bancos de dados relacionais.

Dentro de um banco de dados, as informações são organizadas em tables(entidades), com suas linhas e colunas, cada tabela possui os seus campos (títulos das informações), e suas informações, que preenchem cada campo. Ex: Campo Nome, possui o Guilherme, Joao, Maria (informações).

Existem algumas regras para nomearmos tabelas e campos.

  • Deve começar por uma letra do alfabeto.
  • Não se deve colocar caracteres especias: ()+-/* etc....
  • Não pode conter espaços

Tipos de Campos no SQL.

  • text
  • number
  • Datatime
  • Primary Key

Comandos Básicos:

  • create database bancoEscola - cria um novo banco de dados.
  • use bancoEscola - seta para o mysql passar a usar o banco desejado.
  • create table nomeTabela - cria uma nova tabela.

Comando Select

Tem por função buscar informações e mostrar para gente.

  • SELECT * FROM nomeTable - vai trazer todos os campos da tabela desejada.
  • SELECT nome, idade FROM nomeTable - vai trazer apenas os campos nome e idade.
  • SELECT * FROM nomeTable WHERE id = 1 - vai trazer apenas o resultado que o id seja igual a 1.
  • SELECT * FROM nomeTable WHERE nome like "j%" - vai trazer quem o nome começa com j.
    • %j - colocaria quem acaba com J.
    • %j% - colocaria quem tem J no meio.

Operadores Relacionais

SQL irá se basear em queries (buscas). Para isso precisamos dominar seus operadores, para criar cada vez mais queries efetivas e precisas.

  • = - igual - WHERE cpf = 183283273. (Só funciona em campo numérico).
  • Like - Funciona igual o = , porém para textos. Textos sempre entre "".
  • > - Maior - WHERE idade > 10.
  • < - Menor - WHERE idade < 10.
  • >= - Maior Igual - Where idade >= 10
  • <= - Menor Igual - Where idade <= 10
  • <> - Não Igual a - Where idade <> 10
  • != - Diferente de -

Operadores Lógicos

  • AND - e - SELECT * FROM aluno WHERE nome like "%j" AND idade > 10
    • Traga os alunos que o nome termine com j E que tenha mais de 10 anos.
  • OR - ou - SELECT * FROM aluno WHERE nome like "%j" OR idade > 10
    • Traga os alunos que o nome termine com j OU que tenha mais de 10 anos.
  • BETWEEN - entre - SELECT * FROM estudantes WHERE id BEETWEEN 4 and 7
    • Traga os estudantes que o ID esteja entre 4 e 7.
    • NOT BETWEEN - Traga todos os estudantes que o ID nao seja entre 4 E 7.
  • IN/NOTIN - SELECT * FROM funcionarios WHERE id_Loja IN (1,2,5)
    • Traga todos os funcionários que o id_Loja sejam 1, 2, 5.
    • NOT IN - Traga todos os funcionários que o id_Loja não sejam 1, 2, 5
  • Is NULL/NOT NULL - SELECT * FROM estudantes WHERE id IS NULL
    • Traga todos os estudantes que o id seja NULL
    • NOT NULL - Traga todos os estudantes que o id não seja NULL.

Inserindo, Atualizando, Deletando

  • INSERT INTO - Adiciona na tabela.
    • INSERT INTO aluno (nome, idade, cpf) VALUES ("Joao", 25, 17171717)
    • Insere no aluno um novo registro com nome de Joao, idade 25, e cpf 17171717.
  • UPDATE - Altera algo na tabela.
    • UPDATE aluno SET nome="Marcos" WHERE id = 2
    • Pega na tabela o id de número 2, e altera seu nome para Marcos
  • DELETE - Deleta algum registro na tabela (não apaga campo, só o registro inteiro).
    • DELETE FROM aluno WHERE id = 3
    • Deleta da tabela aluno o registro em que o id seja 3
